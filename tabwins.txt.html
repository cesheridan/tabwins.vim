 37 Section 1: Introduction                         TabwinsIntroduction
 38 ====================================================================
 39 
 40 Easy access to the power of the Vim GUI.
 41 
 42 See the screen shots at
 43   https://github.com/cesheridan/graphics/tree/master/tabwins
 44 
 45 
 46 1 command for immediate access to customized window structures
 47 and content ~during~ Vim sessions.
 48 
 49 The Developer
 50   - Avoids hit-or-miss, time-consuming, manual, repetitious
 51     runs of :split/:vsplit, and thought-sinks about how
 52     to produce the intended window structure
 53 
 54   - Avoids losing current configurations due to mid-session
 55     reloads of backed-up Vim sessions that have preferred
 56     window structures
 57 
 58   - Avoids loss of focus from these discontinuities
 59 
 60 
 61 FORMS --
 62 
 63   :{Tabwins cmd}   {window_dimensions}; {fill_spec,...};  {tab_option, ...}
 64   :{symmetric cmd}                      {fill_spec,...};  {tab_option, ...}
 65 
 66   Where
 67      :{Tabwins cmd}   is any of :Tabwins, :TabwinsVertical, :TabwinsHorizontal
 68 
 69      :{symmetric cmd} is any of the form :{V|H}IxJ or :{V|H}I
 70 
 71      'fill_spec' is a file, dir, shell/vim var, or string evaluable
 72      by :execute -- see EXAMPLES
 73 
 74      'tab_option' has Vim key:value pair syntax -- see EXAMPLES
 75 
 76   Semicolons separate arg groups, commas separate inside groups.
 77 
 78   {window_dimensions} is the only mandatory arg, for :{Tabwins cmd}
 79 
 80   SECTION TabwinsUsage IN tabwins.txt HAS MORE DETAIL.
 81 
 82 
 83 EXAMPLES --
 84 
 85   >> Empty Windows
 86 
 87   :Tabwins 1211
 88   " For a new tab of 4 columns of 1, 2, 1, & 1 windows successively
 89   " left to right.
 90 
 91   :TabwinsHorizontal 4221
 92   " For a new tab of 4 rows, with 4,2,2, & 1 windows successively
 93   " top to bottom.
 94 
 95 
 96   >> Populated Windows
 97 
 98   :Tabwins            1211; $HOME, '~/.bashrc', '~/.vimrc', '/', '/usr/bin'
 99   " 'fill_content' specs of files are automatically opened via :edit,
100   " while specs of dirs are opened via :Explore
101   " (Specify the '' empty string for windows intended to be empty.)
102 
103   :Tabwins           11111; $HOME, '~/.bashrc', '~/.vimrc', '/', '/usr/bin'
104   " Same buffer content, in different window structure, of 5 cols,
105   " one window in each.
106 
107   :TabwinsHorizontal 11111; $HOME, '~/.bashrc', '~/.vimrc', '/', '/usr/bin'
108   " Flip axis priority of above to the horizontal
109 
110   :Tabwins             312; '~/.bashrc', '~/.vimrc', '~/.git', $HOME, '/', '/usr/bin'
111 
112 
113   >> Symmetric Window Structures
114 
115   :V5x8
116   :H5x8
117   :V8x14
118   :H8x14
119   " Empty windows
120 
121   :V5x1 $HOME, '~/.bashrc', '~/.vimrc', '/', '/usr/bin'
122   " Structure & content same as :Tabwins 11111 above
123 
124   :V5   $HOME, '~/.bashrc', '~/.vimrc', '/', '/usr/bin'
125   " Same result again: since the secondary axis has only 1
126   " window for its dimensions, V5 is available as an even
127   " shorter alias.
128 
129 
130   >> Dimensions >9
131 
132   :Tabwins [ 5, 8, 13, 21 ]
133   " For a 4-column tab with 5, 8, 13, 21 windows successively left to right.
134 
135   :Tabwins [ 14, 14 ]
136   " Alternately, :V2x14
137 
138   :Tabwins [ 40, 30, 40, 30 ]
139   " For large monitors ...
140 
141 
142   >> fill_specs with Multiple Commands
143 
144   :Tabwins 1112;  $HOME, '~/.bashrc', '~/.vimrc', '/', 'Explore /usr/bin | resize 20'
145   " Does a :resize of the last window, via Vim '|' command concat.  If a file
146   " or dir is followed with concated commands, the caller needs to explicitly
147   " add the command that opens the file or dir, i.e. 'Explore' in this example.
148 
149 
150   >> Tab Options
151 
152   :Tabwins 1211;  $HOME, '~/.bashrc', '~/.vimrc', '/', '/usr/bin'; 'ending_window_number' : 2
153   :Tabwins 1211;  $HOME, '~/.bashrc', '~/.vimrc', '/', '/usr/bin'; 'netrw_liststyle'      : 3
154 
155   :Tabwins 1211;  $HOME, '~/.bashrc', '~/.vimrc', '/', '/usr/bin'; 'netrw_liststyle'      : 3, 'ending_window_number' : 2
156   " Multiple tab_options in same command.
157 
158 TABWINS MENU --
159 
160   Default settings load Vim menu Tabwins, for access to selected
161                         ~~~~~~~~~~~~~~~~
162   tab-builder commands.  Those and additional tab builders are
163   available via Vim commands defined in tabwins.vim. The menu has
164   only a small portion of the tab builders defined in tabwins.vim.
165 
166 
167 DEPLOYMENT --
168 
169   tabwins.vim tabwins/ dir is structured for deployment in a
170   pathogen-managed bundle directory, e.g. ~/.vim/bundle/
171 
172   If pathogen is not installed, copy the the files in the tabwins
173   subdirs to the same subdirs in ~/.vim.
174 
175   Enable :h tabwins in a pathogen environment with
176     :Helptags
177   and in a non-pathogen environment with
178     :helptags

182 Section 2: Usage                                       TabwinsUsage
183 ====================================================================
184 
185 => See FORMS    in TabwinsIntroduction
186 => See EXAMPLES in TabwinsIntroduction
187 
188 
189 THREE TABWINS COMMANDS --
190 
191   These commands all take the same arguments.
192 
193   :Tabwins
194     Same as :TabwinsVertical
195 
196   :TabwinsVertical
197     Gives priority to the vertical axis.
198 
199   :TabwinsHorizontal 
200     Gives priority to the horizontal axis.
201 
202   See TabwinsVandH about primary/secondary axes and
203   vertical/horizontal priority.
204 
205   These commands accept arbitrary dimension specification,
206   unconstrained by configuration vars, other than
207   windows-per-tabsize limits of the Vim instance.
208 
209 
210 SYMMETRIC WINDOW STRUCTURES --
211 
212   In these, the lengths of the secondary axis dimensions
213   are all equal.
214 
215   Symmetric commands are available to reduce keystrokes.
216 
217   'H' indicates horizontal priority, 'V' vertical priority.
218 
219   Loading tabwins.vim creates :{V|H}IxJ commands where the
220   set of IxJ is the permutations of:
221     g:tabwins_max_v_size 
222     g:tabwins_max_h_size 
223 
224   The same global vars determine the set of :{V|H}J commands
225   also produced on tabwins.vim load, where N is from 1 to the
226   value of the relevant g:tabwins_max_{h|v}_size var above.
227 
228 
229 DIMENSIONS GREATER THAN 9 --
230 
231   Dimensions >9 are unparsable when {window_dimensions} is
232   an integer.  To enable dimensions >9, use list syntax as
233   shown in EXAMPLES of TabwinsIntroduction
234 
235   List syntax applies to any tab that one or more dimension >9.
236   All the dimensions of such a tab are specified in one list.
237 
238 
239 FILL SPECS WITH MULTIPLE COMMANDS --
240 
241   Multiple Vim commands can be specified in a fill_spec by
242   prepending commands with the Vim '|' concatenation operator.
243   See EXAMPLES in TabwinsIntroduction.
244 
245 
246 TAB OPTIONS --
247 
248   Available tab_options are as below, with defaults on
249   the right.
250 
251   These apply to the entire tab and can be combined when
252   separated by commas.
253 
254      'explore_cmd_name'          : 'Explore',
255      'netrw_liststyle'           : g:tabwins_netrw_liststyle_default,
256      'line_number_at_window_top' : g:tabwins_netrw_line_number_default,
257 
258      => Use a fill_spec for window-specific override of all above
259 
260      'first_window_number'       : 1,
261      => window population starts with this num, so you wouldn't have to
262         have to have a fill_spec for lower-numbered windows.
263 
264      'ending_window_number'      : 1
265      => the active window then window population ends.
266 
267 
268   See EXAMPLES in TabwinsIntroduction.
269 
270   Function Fill_tab() processes tab_options.
271 
272 
273 HELP --
274 
275   :TabwinsFindTabwins will load the tabwins .vim & .txt files
276   into a V2 window, if your Vim &runtimepath can see them.
277 
278   This command is included in the 'Tabwins' menu.

282 Section 3: Vertical & Horizontal                    TabwinsVandH
283 ====================================================================
284 
285 Windows are built one axis at a time. The first axis built
286 is the primary axis, and the other axis is the secondary axis.
287        ~~~~~~~~~~~~                            ~~~~~~~~~~~~~~
288 If the tab starts by building the vertical axis, then the
289 vertical axis is the primary axis, and the horizontal axis
290 is the secondary axis. And vice-versa.
291 
292 What's the difference between Horizontal and Vertical Tabs ?
293 
294   Structurally, V2x3 is the same as H3x2.  The only difference
295   is how Vim numbers the tabs, based on the axis that is the
296   primary axis.
297 
298   IF the primary axis is vertical, then numbering starts in
299   col 1 and goes down the windows in col 1, then to the top
300   of col 2, then down col 2, and so on.
301 
302   IF the primary axis is horizontal, numbering starts in
303   row 1 and goes left to right, then to row 2, left to right,
304   and so on.
305 
306   Most of the asymmetric tab builders defined in the plugin
307   use a vertical primary axis.  Developers can define any
308   mix of primary axis in their own tab builders -- See
309   TabwinsAddingNewBuilders

313 Section 4: 'Tabwins' Menu                               TabwinsMenu
314 ====================================================================
315 
316 In the 'Tabwins' menu built on plugin load, the extent that
317 populated tabs fill their buffers is dependent on the shell
318 vars available to the Vim session, and files/dirs present in
319 the local environment.
320 
321 The menu items reference some of these shell vars.
322 
323 The developer is encouraged to customize this menu for local usage.
324 
325 Update or replace func
326   function Tabwins_menu_build()
327 in tabwins.vim to do so.

331 Section 5: Configuration                      TabwinsConfiguration
332 ====================================================================
333 
334 The below globals are defined in tabwins.vim and are overridable.
335 
336 To limit tabwins.vim load to once per Vim session:
337   let g:tabwins_multiple_loads_are_permitted = 'N'
338 Default: 'Y'
339 
340 To prevent 'Tabwins' menu addition when tabwins loads:
341   let g:load_tabwins_menu_is_wanted          = 'N'
342 Default: 'Y'
343 
344 To change the 'Tabwins' menu number:
345   let g:tabwins_menu_number                  = {menu_number}
346 Default: 9998
347 
348 To change the default top-line of netrw windows:
349   let g:tabwins_netrw_line_number_default    = {N}
350 Default: g:tabwins_netrw_line_number_dirpath = 3
351 Calls to Fill_tab() can override this on a per-tab basis.
352 
353 To change the default netrw liststyle:
354   let g:tabwins_netrw_liststyle_default      = {N}
355 Default: g:tabwins_netrw_liststyle_default   = 1
356 Calls to Fill_tab() can override this on a per-tab basis.
357 if netrw var g:netrw_liststyle is not defined, it gets this value.
358 
359 To change Vertical/Horizontal max dimensions:
360   let g:tabwins_max_v_size                   = {N}
361   let g:tabwins_max_h_size                   = {N} 
362 Default: 15 for each

366 Section 6: Adding New Builders            TabwinsAddingNewBuilders
367 ====================================================================
368 
369 
370 ADDING SYMMETRIC TAB COMMANDS --
371 
372   Bump either/both values of:
373     g:tabwins_max_v_size
374     g:tabwins_max_h_size
375 
376   above the values assigned in tabwins.vim.
377 
378   Loading tabwins.vim will build commands for all the HxV and
379   VxH permutations.
380 
381   Of course, the set of symmetric tab builder commands can be reduced by
382   lowering the values of the same globals.
383

385 MORE GENERALLY --
386 
387   Create your own Vim commands from the various : commands in
388   wintabs.vim.  See the plugin code for examples.
