 35 Section 1: Introduction                         TabwinsIntroduction
 36 ====================================================================
 37 
 38 tabwins.vim creates and enables definition of Vim commands that
 39 BUILD TABS with:
 40 
 41   1) SYMMETRIC  window structures of EMPTY BUFFERS 
 42 
 43      e.g. :V4x2 for 4 columns, each with 2 windows
 44 
 45 
 46   2) ASYMMETRIC window structures of EMPTY BUFFERS
 47 
 48      e.g. :V2113 for 4 columns, of 2, 1, 1, & 3 windows respectively
 49 
 50 
 51   3) EITHER of the above structures, with BUFFERS POPULATED
 52      by files, dirs, or results of running :execute on
 53      user-specified strings.
 54 
 55      e.g. :Open_tab_unix_filesystem_1 for an asymmetric V234 tab
 56      populated with typical unix dirs.
 57 
 58 Loading tabwins.vim creates vim commands that have name formats
 59 as described in TabwinsUsage.
 60 
 61 Default settings load Vim menu Tabwins, for access to selected
 62                       ~~~~~~~~~~~~~~~~
 63 tab-builder commands.  Those and additional tab builders are
 64 available via vim commands defined in tabwins.vim. The menu has
 65 only a small portion of the tab builders defined in tabwins.vim,
 66 and the user is encouraged to add tab builder commands as needed.
 67 
 68 By enabling customized tab creation ~during~ sessions, tabwins.vim
 69 gives Vim users immediate access to the window structures and
 70 content that they most commonly use.  The Vim user is saved from
 71 having to reload backed up Vim sessions, and the consequent
 72 loss of current configuration, and the loss of user focus that
 73 follows.
 74 
 75 Easy access to the power of the Vim GUI.
 76 
 77 
 78 This plugin is structured for deployment in a pathogen-managed
 79 bundle directory.
 80 
 81 See the remainder of tabwins.txt for more information.

 85 Section 2: Usage                                       TabwinsUsage
 86 ====================================================================
 87 
 88   See TabwinsVandH for background about Vertical/Horizontal
 89   and primary/secondary axes.
 90 
 91   Loading tabwins.vim creates vim commands that have name formats
 92   as described below.
 93 
 94 
 95   WINDOWS OF EMPTY BUFFERS, SYMMETRIC & ASYMMETRIC
 96 
 97   1) :V{N} & :H{N}   -- e.g. :V3 & :H4
 98 
 99     One-dimensional tabs, w/ primary axis identifed by V or H, and
100     w/ N windows.
101 
102     E.g. :V3 creates a tab of 3 vertical windows, and :H4 creates
103     a tab of 4 horizontal windows.
104 
105     Loading tabwins.vim creates :HN & :VN commands where N is from
106     1 to the value of the relevant axis size in:
107        g:tabwins_max_v_size
108        g:tabwins_max_h_size
109 
110 
111 
112   2) :V{I}x{J} & :H{I}x{J}  -- e.g. :V3x2 & :H4x3
113 
114     Two-dimensional tabs, w/ primary axis identifed by
115     V or H.
116 
117     E.g. V3x2 means a tab of 3 vertical spaces, and inside
118     each vertical space, 2 horizontal windows. Whereas H4x3
119     means a tab of 4 rows, each w/ 3 windows.
120 
121     The window structures in group 1) are all in
122     this group, but here with names like V5x1 rather
123     than the simpler V5.
124 
125     Loading tabwins.vim creates :VIxJ & :HIxJ commands where
126     the set of {I}x{J} is the permutations of:
127        g:tabwins_max_v_size
128        g:tabwins_max_h_size
129 
130 
131 
132   3) :V{J...} & :H{J...}  --  e.g. :V1321 & :H122 
133 
134     Tabs w/ primary axis identifed by V or H.  Each integer
135     specifies the number of windows inside each successive vertical
136     or horizontal space.
137 
138     E.g. V23 means a tab w/ 2 vertical spaces, the left having
139     2 horizontal windows, and the right having 3 horizontal windows.
140     Whereas H14 means two rows, the top w/ 1 window and the
141     lower row w/ 4 windows.
142 
143     Loading tabwins.vim creates the V{J...} & H{J...} commands
144     defined by functions
145       Create_tab_v*() and
146       Create_tab_h*().
147 
148 
149   POPULATED WINDOWS
150 
151   These can be both Symmetric and Asymmetric, for both V and H priority
152   tabs.  The user specifies the content that fills each window buffer.
153   See the examples in the Tabwins menu, as well as documentation in
154   TabwinsAddingNewBuilders.
155 
156 
157   OPERATING SYSTEMS
158 
159   Core code has run successfully for gvim on Linux, Sun Solaris, HPUX,
160   iMac OS X, MSWIN, & cygwin.
161 
162   On MSWIN, demos of populated tabs may fail to load some buffer
163   content due to MSWIN file system syntax conventions.

167 Section 3: Vertical & Horizontal                    TabwinsVandH
168 ====================================================================
169 
170 Windows are built one axis at a time. The first axis built
171 is the primary axis, and the other axis is the secondary axis.
172        ~~~~~~~~~~~~                            ~~~~~~~~~~~~~~
173 If the window starts by building the vertical axis, then the
174 vertical axis is the primary axis, and the horizontal axis
175 is the secondary axis. And vice-versa.
176 
177 What's the difference between Horizontal and Vertical Tabs ?
178 
179   Structurally, V2x3 is the same as H3x2.  The only difference
180   is how vim numbers the tabs, based on the axis that is the
181   primary axis.
182 
183   IF the primary axis is vertical, then numbering starts in
184   col 1 and goes down the windows in col 1, then to the top
185   of col 2, then down col 2, and so on.
186 
187   IF the primary axis is horizontal, numbering starts in
188   row 1 and goes left to right, then to row 2, left to right,
189   and so on.
190 
191   Most of the asymmetric tab builders defined in the plugin
192   use a Vertical primary axis.  The Vim user can define any
193   mix of primary axis in their own tab builders -- See
194   TabwinsAddingNewBuilders

197 Section 4: Tabwins Menu                                TabwinsMenu
198 ====================================================================
199 
200 In the Tabwins menu as delivered with this plugin, the
201 extent that populated tabs fill their buffers is dependent
202 on the shell vars available to the Vim session.
203 
204 The menu items document some of these vars.
205 
206 The tab of dot files will be populated to the extent that $HOME
207 holds the specified files.
208 
209 The user is encouraged to customize tab builders to match their
210 usage.
211 
212 Update or replace func
213   Tabwins_menu_build()
214 in tabwins.vim to access any set of tab builders.

218 Section 5: Configuration                      TabwinsConfiguration
219 ====================================================================
220 
221 The below globals are defined in tabwins.vim and are overridable.
222 
223 To prevent 'Tabwins' menu addition when tabwins loads:
224   let g:load_tabwins_menu_is_wanted       = 'N'
225 
226 To change the 'Tabwins' menu number:
227   let g:tabwins_menu_number = {menu_number}
228 
229 To change the default top-line of netrw windows:
230   let g:tabwins_netrw_line_number_default = {N}
231 Calls to Fill_tab() can override this on a per-tab basis.
232 
233 To change the default netrw liststyle:
234   let g:tabwins_netrw_liststyle_default   = {N}
235 Calls to Fill_tab() can override this on a per-tab basis.
236 
237 To change V/H max dimensions:
238   let g:tabwins_max_v_size                = {N}
239   let g:tabwins_max_h_size                = {N}

243 Section 6: Adding New Builders      TabwinsAddingNewBuilders
244 ====================================================================
245 
246 1) ADDING SYMMETRIC TAB COMMANDS  (Empty Buffers)
247 
248 Bump either/both values of:
249   g:tabwins_max_v_size
250   g:tabwins_max_h_size
251 above the values assigned in tabwins.vim. Loading tabwins.vim will
252 build commands for all the permutations of the size vars.
253 
254 Of course, the set of symmetric tab builder commands can be reduced by
255 updating the same globals.
256 
257 
258 
259 2) ADDING ASYMMETRIC TAB COMMANDS  (Empty Buffers)
260 
261 Copy and update any of the
262   Create_tab_v{J...}()    
263   Create_tab_h{J...}()
264 functions.
265 
266 See the comments in the asymmetric command building code in tabwins.vim.
267 building code.

270 
271 3) ADDING COMMANDS FOR TABS OF POPULATED BUFFERS
272 
273 Update, clone, or replace any of the
274   Open_tab_*()
275 functions in tabwins.vim to create populated window structures.
276 
277 Example in tabwins.vim: 
278 
279       " ---------------------------------------------------------------
280       function! Open_tab_home_dir()
281       " ---------------------------------------------------------------
282         "1ST, Create the windowed-tab
283         :V232
284         "VERTICAL ASYMMETRIC TAB.  3 cols, from left to right with
285         "2, 3, and then 2 windows successively.
286 
287         let l:home_parent_dirpath = substitute(finddir($HOME), '\/\w\+$', '', 'g')
288 
289         "2ND, Fill buffers in the new tab
290         call Fill_tab({
291         \ 'line_number_at_window_top' : 1,
292         \ 'window_fill_specs' : [
293         \   $HOME,
294         \   'Explore' . l:home_parent_dirpath,
295         \
296         \   'edit! ~/.bashrc',
297         \   '~/.gitignore',
298         \   'enew!',
299         \
300         \   '~/.vimrc',
301         \   '~/.vim',
302         \ ]
303         \})
304         "Fill_tab() fills windows in the order of elements
305         "in 'window_fill_specs' You want to spec something for each
306         "of the windows created by :V1321 above,
307         "either commands, filepaths, or dirpaths.
308         "Specs can include shell or vim vars, and if files or dirs
309         "the spec can include a vim command or use the default :edit or
310         ":Explore call in Fill_tab()
311 
312         "3d, optional, apply Vim resize and/or other commands to the windows/buffers.
313         2wincmd w
314         resize 30
315 
316         1wincmd w
317         "And end back in window 1
318       endfunction
319 
320 
321 Every tab-builder calls Fill_tab(), with signature: 
322 
323       'explore_cmd_name'          : 'Explore',
324       'window_fill_specs'         : [],
325       'netrw_liststyle'           : g:tabwins_netrw_liststyle_default,
326       'line_number_at_window_top' : g:tabwins_netrw_line_number_default,
327       'first_window_number'       : 1,
328       'ending_window_number'      : 1,
329 
330 See the Open_tab_*() functions for examples of overriding Fill_tab()
331 defaults.
332 
333 See the NOTE: comments in tabwins.vim for more information. Note particularly
334 the directives about window deletion order in calls to Close_windows().
